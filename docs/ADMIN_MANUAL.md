## 1. 시스템 개요

### 학생용 흐름 (담당자가 직접 할 일 없음)
1. 학생이 우산에 붙은 QR 을 스마트폰으로 스캔
2. 자동으로 대여 화면이 뜸 → 학번·이름·전화번호 입력 → **대여 완료**
3. 반납할 때 **같은 QR 을 다시 스캔** → 반납 화면이 뜸 → 학번·이름 입력 + 거치대에 걸린 우산 사진 촬영 → **반납 완료**

> **중요**: QR 은 우산당 1개뿐입니다. 우산 상태(대여 가능/대여 중)에 따라 같은 QR 이 자동으로 대여 화면 또는 반납 화면을 보여줍니다. 대여용·반납용이 따로 있는 것이 아닙니다.

### 반납 원칙
- **다음날(익일) 반납**이 원칙입니다.
- 다음날이 주말(토·일)이나 공휴일이면, **가장 빠른 평일**에 반납합니다.
- 예: 금요일 대여 → 월요일 반납. 수요일(공휴일 전날) 대여 → 목요일이 공휴일이면 금요일 반납.

### 담당 학생의 역할
- 우산 추가·제거·점검 상태 관리
- QR 코드 인쇄 및 새 우산에 부착
- 장기 미반납자 연락
- 주기적인 대여 현황 점검
- 반납 사진 확인 (파손·분실 여부)

## 2. 로그인과 로그아웃

### 접속 주소
```
https://umbrella.project-ms.kr/admin
```

> 주소는 변경될 수 있으니 인수인계 시 현재 주소를 재확인하세요.

### 로그인
1. 위 주소에 접속하면 로그인 화면이 뜹니다.
2. 인수인계받은 **아이디 / 비밀번호** 입력
3. "로그인" 버튼 클릭 → 대시보드로 이동

### 로그아웃
- 왼쪽 사이드바 맨 아래 **로그아웃** 버튼 클릭

### 세션 유지 시간
- 로그인 후 **2시간** 동안 세션이 유지됩니다.
- 2시간이 지나면 자동으로 로그아웃되니 다시 로그인해주세요.

### 비밀번호 분실 / 변경
- 비밀번호는 서버에 해시로 저장되어 있어 담당자 본인도 복구할 수 없습니다.
- **변경이 필요한 경우 시스템 개발자에게 요청**하세요. 절대 SNS/카카오톡으로 비밀번호를 공유하지 마세요.

## 3. 대시보드 읽는 법

로그인 후 처음 보이는 화면입니다. 메뉴는 왼쪽 사이드바에 있습니다.

### 상단 요약 카드 (4개)
| 카드 | 의미 |
|---|---|
| **전체 우산** | 시스템에 등록된 모든 우산 수 |
| **대여 가능** | 지금 빌릴 수 있는 우산 수 |
| **대여 중** | 지금 누군가 쓰고 있는 우산 수 |
| **장기 미반납** | 반납 기한을 넘긴 우산 수 |

### 주간 차트
- 최근 7일 동안의 **일별 대여·반납 추이**를 그래프로 보여줍니다.
- 비 오는 날 대여가 급증하는지 한눈에 파악 가능.

### 장기 미반납 목록
- 반납이 늦어지는 대여 기록이 우선순위 순으로 나열됩니다.
- 학번·이름·전화번호·대여일이 표시되어 바로 연락 가능.

## 4. 우산 등록 · 삭제 · 상태 변경

왼쪽 사이드바의 **우산 관리** 메뉴로 이동.

### 새 우산 추가
1. 우측 상단 **"+ 우산 추가"** 버튼 클릭
2. 우산 번호 입력 (예: `006`, `007` …)
   - 숫자 3자리 형식 권장 (`001` ~ `999`)
   - 번호는 **중복 불가**, 기존 번호와 겹치면 에러
3. "추가" 버튼 클릭 → 목록에 새 우산이 나타납니다.
4. [5장](#5-qr-코드-발급과-부착) 을 참고해 QR 을 인쇄하여 우산에 부착.

### 우산 상태 변경
각 행의 **상태 변경** 드롭다운에서 선택:
- **대여가능** — 학생이 빌릴 수 있는 상태 (기본)
- **대여중** — 누군가 사용 중 (보통 자동 변경, 수동은 거의 안 씀)
- **점검중** — 파손·수리·일시 회수된 우산. 이 상태에선 대여 QR 스캔 시 "점검 중" 안내가 뜨고 대여 불가.

### 우산 삭제
- 각 행 오른쪽 🗑️ 아이콘 클릭
- 확인 대화상자에서 "확인" 클릭
- **주의**: 과거 대여 기록이 있는 우산을 삭제하면 통계에서 사라집니다. 가능하면 **점검중 상태로 두는 것** 을 권장.

## 5. QR 코드 발급과 부착

### QR 발급
1. **우산 관리** 페이지의 해당 우산 행에서 **"QR 보기"** 클릭
2. 팝업에 QR 코드 이미지와 URL 이 표시됩니다.
3. **PNG 다운로드** 버튼으로 이미지 저장 (`umbrella-001.png` 등)
4. A4 용지나 스티커지에 인쇄

### QR 부착 가이드
- **권장 크기**: 4 × 4 cm 이상 (스마트폰이 쉽게 인식)
- **재질**: 방수 스티커 또는 투명 테이프 코팅
- **위치**: 우산 손잡이 근처 또는 우산 대 중간부 (학생 눈높이에 잘 보이는 곳)
- **함께 부착할 것**: 우산 번호 (큰 글씨) + 유의사항 스티커 (`UMBRELLA_NOTICE` 참고)

### QR 1개로 대여·반납 모두 처리
- **같은 QR 이 우산의 현재 상태에 따라 자동으로 대여/반납 화면을 보여줍니다.**
- 우산당 QR 은 1장만 붙이면 됩니다.
- 학생에게 "빌릴 때 스캔, 돌려줄 때 다시 스캔" 이라고만 안내하면 됩니다.

## 6. 대여 현황 확인

왼쪽 사이드바의 **대여 관리** 메뉴.

### 필터
상단 필터에서 상태별로 볼 수 있습니다:
- **전체** — 모든 기록
- **대여중** — 아직 반납 안 된 건
- **반납완료** — 완료된 건
- **장기미반납** — 기한 초과된 건
- **분실** — 담당자가 분실 처리한 건

### 검색
학번 또는 이름으로 검색 가능. 예: `2024` 입력하면 2024학번 전체가 필터됨.

### 개별 기록 클릭
행을 클릭하면 상세 정보가 열립니다:
- 대여 시각 / 반납 시각
- 반납 사진 (반납 완료 건만)
- 수동으로 **상태 변경** (분실, 반납 완료 등)
- **메모** 필드: 학생에게 전화한 내용 등을 기록 (예: "2024-05-10 전화 통화, 월요일 반납 약속")

## 7. 장기 미반납 처리

### 장기 미반납 판정
- 대여일로부터 **일정 기간이 지나도 반납되지 않은 경우** 시스템이 자동으로 "장기미반납" 상태 배지를 붙입니다.
- 대시보드 상단 카드와 **장기 미반납 목록** 에 표시됩니다.

### 대응 절차 (권장)
1. **1차 연락 (대여 후 2일 경과 시점)**: 전화번호로 문자 또는 통화
2. **2차 연락 (4일 경과)**: 직접 만나서 안내
3. **3차 (1주일 이상)**: 분실로 간주, **분실 상태로 변경**하고 메모에 경위 기록
4. 필요시 지도교사 또는 학생부에 인계

### 메모 작성 팁
- `2025-05-10 전화 통화, 월요일 반납 약속`
- `2025-05-13 본인 반납 확인, 우산 손잡이 약간 휨 (사용 가능 수준)`
- 메모는 다음 담당자에게 그대로 인계되므로 **짧지만 누가 봐도 이해되게**.

## 8. 데이터 내보내기 (CSV)

왼쪽 사이드바의 **내보내기** 메뉴.

### 용도
- 학기말 결산 보고서
- 파손·분실 통계
- 학생회/학교 행정 문서용 자료

### 다운로드 방법
1. 기간 선택 (예: "이번 학기", "최근 30일")
2. **"CSV 다운로드"** 버튼 클릭
3. Excel 로 열어서 그대로 활용 가능 (한글 인코딩 자동 처리)

### CSV 필드
- 대여 ID, 우산 번호, 학번, 이름, 전화번호, 대여일, 반납일, 상태, 반납 사진 URL, 메모

## 9. 반납 사진 확인

반납 시 학생이 업로드한 사진은 대여 기록 상세 페이지에서 확인할 수 있습니다.

### 확인 포인트
- **우산이 거치대에 제대로 걸려 있는지** (바닥에 두고 사진만 찍은 경우 재반납 요구)
- **파손 여부** (뼈대 휨, 천 찢어짐 등)
- 파손이 심하면 해당 우산을 **점검중** 상태로 전환 후 수리/교체

### 사진 자동 삭제
- 반납 완료 후 **7일이 지난 사진은 자동으로 삭제**됩니다 (기본 설정).
- 개인정보 보호 및 저장 비용 절감 목적.
- 파손 증거로 보관해야 할 경우 담당자가 **직접 이미지를 다운로드**하여 별도 저장.

## 10. 자주 발생하는 상황

### Q1. 학생이 "QR 스캔해도 화면이 안 뜬다" 고 해요
- 대부분 **스마트폰 카메라 앱이 아닌 다른 앱**으로 스캔해서 그렇습니다.
- iPhone: 기본 **카메라 앱** 으로 스캔 → 자동 링크 뜸
- Android: **기본 카메라** 또는 **네이버/카카오 QR** 사용
- QR 이 오염되었거나 찢어졌다면 즉시 재인쇄.

### Q2. 반납할 때 "대여 정보를 찾을 수 없습니다" 에러
- 학생이 **학번·이름을 대여할 때와 다르게** 입력했기 때문입니다 (공백, 한자 혼용, 띄어쓰기).
- 학생에게 정확히 재입력 요청.
- 그래도 안 되면 **대여 관리** 에서 해당 우산의 대여 기록을 찾아 수동으로 반납 처리.

### Q3. 우산만 돌려놓고 반납 처리를 안 한 학생이 있어요
- 우산 상태가 **여전히 "대여 중"** 으로 남아있습니다.
- 해당 대여 기록을 찾아 **수동 반납 처리** 후, 메모에 "2025-xx-xx 직접 반납 확인, 학생 본인은 QR 미스캔" 기록.

### Q4. 점검을 위해 일부 우산을 회수하고 싶어요
- 해당 우산들의 상태를 **점검중** 으로 변경.
- 스캔해도 "점검 중" 안내만 뜨고 대여 불가.
- 수리 완료 후 **대여가능** 으로 복구.

### Q5. 로그인이 안 돼요
- 대소문자 구분하여 입력했는지 확인.
- 여러 번 실패해도 계정이 잠기지는 않음.
- 진짜 틀린 게 아니라면 시스템 개발자에게 비밀번호 재설정 요청.

### Q6. 사이트가 안 열려요 / 느려요
- 시스템 개발자에게 연락. 서버(VPS) 문제일 수 있음.
- 임시 조치로 **대여·반납 기록을 종이에 받아두었다가** 복구 후 수동 입력.
